# Orchard iPhone 线框与组件拆分

本文件是 [IPHONE_FUNCTIONAL_DESIGN.md](/Users/owen/.codex/worktrees/08f0/Orchard/docs/IPHONE_FUNCTIONAL_DESIGN.md) 的实现级补充，用于说明页面线框、模块结构和 SwiftUI 组件映射。

## 1. 导航结构

```text
TabView
├── 概览 Dashboard
│   └── 任务详情 / 设备详情
├── 任务 Tasks
│   ├── 新建任务 Sheet
│   └── 任务详情
└── 设备 Devices
    └── 设备详情
```

设置不放在底部 Tab 中，通过各页右上角 `gearshape` 进入。

## 8. 当前代码文件映射

- `RootView.swift`
  - App 根入口，负责 `TabView` 和设置弹层
- `DashboardView.swift`
  - 概览页
- `TasksView.swift`
  - 任务页、任务详情、新建任务
- `DevicesView.swift`
  - 设备页、设备详情
- `CompanionSupport.swift`
  - 通用组件、展示文案、状态颜色和设置页
- `AppModel.swift`
  - 全局数据拉取与任务操作

---

## 9. 下一轮可继续拆分

如果继续往产品化推进，下一轮建议做：

1. 把 `TaskDetailView` 和 `CreateTaskSheet` 从 `TasksView.swift` 拆成独立文件
2. 给概览页增加失败任务优先排序和摘要规则
3. 给任务和设备加搜索
4. 引入实时刷新或 WebSocket 推送
5. 为 iPhone 和 Mac 端分别细化布局差异
